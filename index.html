公告：这是一个自建网站！

这是什么网站？用来干嘛？

我们通过GitHub+Hexo，成功在GitHub的服务器上部署了一个博客，用于为大家发布习题和题解，同时，我们也会在这个网站上上传各类学习资源。该网站将会不断进行更新，同时，也希望数字媒体技术专业的各位同学能够积极参与进学习中来。

各位可以做些什么？

如果解出了发布的题目，大家也可以选择自己来写题解，并通过邮箱(3181457165@qq.com)将Markdown格式的题解发给我们，我们会在网站上登出你的题解！同时，如果你在研究一些技术，你同样可以在邮箱发送你的成果，我们可以将你的成果展示在大家面前，形成良好的学术交流环境，或许，在这里，你会有什么新的思路！

在这里，我们也会通过Announcementtag公布后续的促学计划：Markdown教程、Linux系统使用等大量内容都会在这里更新！或许，在这里自学也不失为一种选择！

hexo-bilibili-bangumi

介绍

为 Hexo 添加哔哩哔哩/Bangumi追番/追剧页面，参考自hexo-douban.

Demo

使用

  1. 在hexo generate或hexo deploy之前使用hexo bangumi -u命令更新追番数据，使用hexo cinema -u命令更新追剧数据！
  2. 删除数据命令:hexo bangumi -d/hexo cinema -d

获取 Bilibili uid

登录哔哩哔哩后前往https://space.bilibili.com/页面，网址最后的一串数字就是 uid

需要将追番列表设置为公开！

获取 Bangumi id

登录Bangumi后打开控制台(Ctrl+Shift+J)，输入CHOBITS_UID回车，下面会输出id

示例

手动添加番剧/追剧数据

因为某些番剧在哔哩哔哩上没有，但是又想在hexo中展示，怎么办呢？现在支持手动添加番剧数据了！

另外除了 watchedExtra 数组，还可以在后面添加新的数组，可用数组名如下:

可用数组名 含义
wantWatchExtra 想看
watchingExtra 在看
watchedExtra 看过

多主题兼容

  1. Fork此项目并克隆到本地；
  2. 进入项目目录并安装依赖npm install;
  3. 在src/lib/templates/theme/目录内添加主题.css文件（例：butterfly.css）；
  4. 在主题.css文件内添加主题兼容样式表；
  5. 运行命令npm run build;
  6. 提交PR.
